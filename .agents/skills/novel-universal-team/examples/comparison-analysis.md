# 📊 v1 vs v2 비교 분석 — 소설 창작 에이전트 팀

> **작성일:** 2026-05-03  
> **목적:** 기존 에이전트 팀(v1)과 범용 에이전트 팀(v2)의 구조적 차이를 분석하고, 각 시스템의 적합 시나리오를 도출

---

## 1. 아키텍처 비교

### 1.1 구조 다이어그램

**v1 (novel-team-agent):**
```
                 팀 디렉터
                    │
         ┌──── 듀얼 모드 ────┐
         │                   │
    모드1: 집필 보조      모드2: 브레인스토밍
    ┌─────────────┐    ┌─────────────┐
    │ 플롯 메이커   │    │ 아이디어 플래너│
    │ 캐릭터 빌더   │    │ 세계관 설계자  │
    │ 분위기 연출가  │    │ 캐릭터 크리에이터│
    │ 대화 디자이너  │    │ 갈등 디자이너  │
    │ 리뷰어       │    │ 독자 분석가   │
    └─────────────┘    └─────────────┘
```

**v2 (novel-universal-team):**
```
              Harness Layer
                    │
         ┌──── 범용 팀 ────┐
         │                 │
    Core 5인          장르 모듈 (Plugin)
    ┌──────────┐    ┌──────────────────┐
    │ Plotter  │    │ Fantasy          │
    │ Character│    │ Modern Fantasy   │
    │ Stylist  │    │ SF               │
    │ Reviewer │    │ Fusion           │
    │ Editor   │    │ Romance          │
    └──────────┘    │ Thriller         │
                    └──────────────────┘
```

### 1.2 핵심 구조 차이

| 차원 | v1 | v2 |
|------|----|----|
| **에이전트 수** | 5인 (모드별 역할 전환) | 5인 (고정) + 6 모듈 (플러그인) |
| **모드 전환** | 명시적 듀얼 모드 (집필/브레인스토밍) | 단일 파이프라인 + 브레인스토밍 변형 |
| **장르 대응** | 내부 밀도 가중치 테이블 | 외부 모듈 플러그인 |
| **품질 게이트** | Reviewer 1단 | Reviewer → Editor 2단 |
| **운영 레이어** | 세션 핸드오프 프로토콜 | Harness Layer (상태/오류/비용) |

## 2. 역할 매핑

| 기능 | v1 역할 | v2 역할 |
|------|---------|---------|
| 플롯 설계 | 플롯 메이커 / 아이디어 플래너 | Plotter Agent |
| 캐릭터 관리 | 캐릭터 빌더 / 캐릭터 크리에이터 | Character Agent |
| 문체·분위기 | 분위기 연출가 | Stylist Agent |
| 대화 설계 | 대화 디자이너 | Character Agent (통합) |
| 검증 | 리뷰어 | Reviewer Agent |
| 문장 다듬기 | (리뷰어에 포함) | **Editor Agent (분리)** |
| 세계관 설계 | 세계관 설계자 (모드2) | 장르 모듈 (Fantasy/SF 등) |
| 갈등 설계 | 갈등 디자이너 (모드2) | Plotter Agent + Thriller 모듈 |
| 독자 분석 | 독자 분석가 (모드2) | Reviewer Agent (시장성 검토) |

### 주요 차이점

1. **대화 디자이너 통합:** v1은 대화 전담 에이전트가 있으나, v2는 Character Agent가 대화까지 담당. 대신 Editor가 문장 수준 폴리싱을 전담.
2. **Editor 신설:** v1은 Reviewer가 검증+다듬기를 겸하나, v2는 검증(Reviewer)과 다듬기(Editor)를 분리하여 2단 게이트 구현.
3. **세계관/갈등 외부화:** v1의 브레인스토밍 모드 전용 역할(세계관 설계자, 갈등 디자이너)이 v2에서는 장르 모듈로 외부화.

## 3. 장르 대응 방식 비교

### v1: 밀도 가중치 테이블

```
| 장르 | 감각 묘사 | 내면 독백 | 대화 비중 | 액션 밀도 |
|------|---------|---------|---------|---------|
| 현대판타지 | ★★★★ | ★★★ | ★★★ | ★★ |
| 법정스릴러 | ★★ | ★★★★★ | ★★★★ | ★ |
```

**장점:** 간결, 빠른 적용  
**단점:** 장르 특화 규칙(마법 시스템, 은폐 메커니즘 등)을 담지 못함

### v2: 장르 모듈 플러그인

```
Fantasy Module → 마법 시스템 설계 프레임워크, 세계관 물리법칙 체크리스트
Thriller Module → 정보 비대칭 맵, 서스펜스 설계 도구, 반전 배치 가이드
```

**장점:** 장르별 심층 규칙 적용 가능, 새 장르 추가 용이  
**단점:** 호출 오버헤드, 모듈 간 충돌 가능성

## 4. 파이프라인 비교

### v1 파이프라인
```
플롯 메이커 → 캐릭터 빌더 → 분위기 연출가 → 대화 디자이너 → 통합 집필 → 리뷰어
                                                                    ↑ 🔴 시 반복
```
- **5단계** (통합 집필 포함 6단계)
- 리뷰어가 최종 관문

### v2 파이프라인
```
Plotter → Character → Stylist → 통합 집필 → Reviewer → Editor
              ↑                                  ↑ 🔴 시 반복
         장르 모듈 (선택적 호출)
```
- **6단계** (Editor 추가)
- 장르 모듈이 중간에 삽입 가능
- Reviewer → Editor 2단 검증

## 5. 강점/약점 분석

### v1 강점
| 항목 | 설명 |
|------|------|
| **전문성** | 한국어 웹소설에 최적화된 규칙과 템플릿 |
| **듀얼 모드** | 기획(브레인스토밍)과 집필을 명확히 분리 |
| **대화 전담** | 대화 디자이너가 서브텍스트·음성 분리에 집중 |
| **외부 스킬 연동** | novel-creation/writing/proofreading-agent와 유기적 연결 |
| **검증된 실적** | 50화+ 연재 실전 투입 경험 (퇴근후 이계상담소 등) |

### v1 약점
| 항목 | 설명 |
|------|------|
| **장르 한정** | 밀도 가중치만으로는 장르 특화 규칙 부족 |
| **단일 검증** | Reviewer 1인이 검증+다듬기 겸임 → 부담 과중 |
| **상태 관리** | 세션 핸드오프가 수동적, 오류 복구 체계 부재 |

### v2 강점
| 항목 | 설명 |
|------|------|
| **범용성** | 장르 불문 적용 가능, 새 장르 모듈 추가 용이 |
| **2단 검증** | Reviewer(구조·설정) + Editor(문장·가독성) 분리 |
| **운영 체계** | Harness Layer로 상태·오류·비용 통합 관리 |
| **모듈러 확장** | 장르 모듈을 독립적으로 추가·수정·제거 가능 |

### v2 약점
| 항목 | 설명 |
|------|------|
| **대화 전담 부재** | Character Agent가 대화까지 겸임 → 대화 품질 리스크 |
| **실전 미검증** | 아직 실전 투입 이력 없음 |
| **호출 복잡도** | 장르 모듈 호출 시점 판단이 오케스트레이터에 의존 |
| **외부 스킬 연동 약화** | v1의 4개 스킬 생태계 대비 독립적 → 기존 자산 활용도 낮음 |

## 6. 적합 시나리오

| 시나리오 | 추천 시스템 | 이유 |
|---------|-----------|------|
| 한국어 웹소설 연재 (단일 장르) | **v1** | 검증된 실전 경험, 전문화된 규칙 |
| 다장르 동시 집필 | **v2** | 장르 모듈 전환으로 유연 대응 |
| 장르 혼합 실험 (무협+SF 등) | **v2** | Fusion 모듈의 톤 블렌딩 |
| 기존 50화+ 작품 벌크업 | **v1** | 시나리오 E(벌크업) 특화 |
| 신규 작품 기획·브레인스토밍 | **v1 또는 v2** | v1: 듀얼모드, v2: 장르모듈로 각각 가능 |
| 비한국어/비웹소설 창작 | **v2** | 범용 설계가 유리 |

---

## 7. 향후 통합 가능성

두 시스템은 현재 독립적으로 운영되지만, 장기적으로 다음과 같은 통합이 가능하다:

1. **v2에 v1의 대화 디자이너 역할 추가** → 대화 품질 리스크 해소
2. **v1에 v2의 Editor Agent 도입** → 2단 검증 체계 강화
3. **v1에 v2의 Harness Layer 적용** → 운영 안정성 향상
4. **v2에서 v1의 기존 스킬(novel-writing-agent 등) 참조** → 기존 자산 활용
