# 📚 장르소설 문체 가이드 (Novel Style Guide)

> **버전:** 1.0  
> **최종 수정:** 2026-04-12  
> **역할:** 다른 소설 관련 에이전트(`novel-writing-agent`, `novel-proofreading-agent`)가 참조하는 **문체 레퍼런스 데이터 허브**  
> **직접 실행:** 이 스킬은 단독으로 실행되지 않는다. 다른 에이전트 스킬에서 `resources/` 하위 파일을 참조한다.

---

## 1. 이 스킬의 목적

장르소설의 문체 품질을 **객관적이고 재현 가능한 기준**으로 관리하기 위한 데이터 허브이다.

"좋은 문장"에 대한 모호한 감상이 아니라, **구체적인 규칙과 예시**를 통해 AI 에이전트가 일관된 품질의 원고를 생산하고 교정할 수 있도록 한다.

---

## 2. 리소스 구조

```
novel-style-guide/
├── SKILL.md                          ← 이 파일 (진입점)
├── scripts/
│   └── ai_sanitizer.py               ← [신규] 초고정밀 AI 문체 소독 검사 엔진
└── resources/
    ├── genre-tone-matrix.md          ← 장르별 어조·시점·비유 매트릭스
    ├── forbidden-expressions.md      ← 금지 표현·Word Spam 패턴 DB
    ├── sound-effects-guide.md        ← 의성어/의태어 표준 DB
    ├── system-message-templates.md   ← 시스템 메시지 포맷 가이드
    ├── volume-assurance-guide.md     ← 유료 연재 분량 확보 실전 가이드
    └── indirect-emotion-lexicon.md   ← [신규] 감정 간접 표현 사전
```

## 3. 리소스 요약 및 참조 지침

### 3.1 장르별 톤 매트릭스 (`genre-tone-matrix.md`)

**언제 참조하는가:**
- 새 작품의 톤앤매너를 설정할 때
- 장르 전환/융합 시 문체 충돌을 방지할 때
- 원고의 어조가 장르 관습에서 이탈했는지 검증할 때

**포함 내용:** 장르 14종에 대한 시점, 서술 톤, 문장 리듬, 비유 관습, 금기 사항

### 3.2 금지 표현 DB (`forbidden-expressions.md`)

**언제 참조하는가:**
- 원고 집필 중 문체 Guard Rail을 적용할 때
- 교정 시 Word Spam 및 톤 이탈을 탐지할 때
- 수식어 밀도, 동의어 나열, 과도한 장음 등을 검증할 때

**포함 내용:** 6개 카테고리의 금지 패턴 + 위반 예시 + 수정 예시 + 감정 간접 표현 3단계 규칙

### 3.3 의성어/의태어 DB (`sound-effects-guide.md`)

**언제 참조하는가:**
- 전투/액션/메카닉/감정 장면의 의성어를 선택할 때
- 의성어 남용 여부를 검증할 때

**포함 내용:** 장면 유형별 추천 의성어·의태어 + 사용 한도

### 3.4 시스템 메시지 포맷 (`system-message-templates.md`)

**언제 참조하는가:**
- 게임판타지, 사이버펑크, 헌터물 등 시스템 UI가 등장하는 작품에서
- 상태창, 경고 메시지, 퀘스트 알림 등을 작성할 때

**포함 내용:** 장르별 시스템 메시지 포맷 템플릿 + 길이 제한 규칙

### 3.5 유료 연재 분량 확보 가이드 (`volume-assurance-guide.md`)

**언제 참조하는가:**
- 집필된 원고의 분량이 5,500자 미만일 때
- 신규 회차 집필 전 분량 확보 전략을 수립할 때
- 보강(벨크업) 작업 시 구체적인 기법을 선택할 때
- 분량 부족 원인을 진단하고 해결법을 찾을 때

**포함 내용:** 7가지 분량 확보 기법 상세 예시, 분량 부족 원인 진단표, 긴급 보강 프로세스, 분량 과다 시 감축 가이드

### 3.6 감정 간접 표현 사전 (`indirect-emotion-lexicon.md`)

**언제 참조하는가:**
- 집필 및 교정 중 인물의 직접적 감정 서술(Tells)을 발견했을 때
- 감정선을 감각적이고 신체적인 물리 반응(Shows)으로 고도화하고자 할 때

**포함 내용:** 5대 감정 축(분노, 슬픔, 공포, 불안/초조, 자괴감/후회)에 대한 직접 서술 vs 간접 묘사 매핑 테이블 및 물리 반응 치환 가이드

## 4. 호출 규칙

다른 에이전트 스킬에서 이 리소스를 참조할 때는 다음 규칙을 따른다:

1. **필요한 리소스 파일만 선택적으로 읽는다.** 전체를 한꺼번에 로드하지 않는다.
2. 리소스의 규칙은 **해당 에이전트의 본문 규칙과 동등한 강제력**을 가진다.
3. 에이전트 본문 규칙과 리소스 규칙이 충돌할 경우, **해당 에이전트 본문이 우선**한다.
4. 문체 교정 및 정량적 검사 시에는 `scripts/ai_sanitizer.py`를 범용 구동한다.
