# Suno AI 곡을 위한 가사(Lyrics) 작성 워크플로우

Suno AI에서 훌륭한 음악을 생성하기 위해서는 단순히 텍스트만 나열하는 것이 아니라, AI가 이해할 수 있는 형태의 가사와 구조 태그(Metatags)를 함께 활용해야 합니다.  
**대상 버전:** Suno v5 (권장) / Studio 1.2+

> [!NOTE]
> v5에서는 가사 정렬(Lyric Alignment)이 크게 개선되어 한국어를 포함한 비영어 가사도 더 정확하게 발음하고 멜로디에 맞춥니다.

---

## 1. 기본 곡 구조(Structure Tags) 설계

AI는 특정한 대괄호 `[ ]` 태그를 인식하여 곡의 단계를 나눕니다. 곡의 전개(기승전결)를 위해 다음 태그들을 활용하여 문단을 분리하세요.

| 태그 | 용도 | 비고 |
|------|------|------|
| `[Intro]` | 곡의 시작 (전주 또는 짧은 허밍/어조) | `[Instrumental Intro]`, `[Ambient Intro]` 변형 가능 |
| `[Verse]` | 스토리 전개, 감정 절제, 템포 유지 | `[Verse 1]`, `[Verse 2]` 등 번호 구분 |
| `[Pre-Chorus]` | 후렴으로 넘어가기 전 긴장감 고조 | 코러스 에너지 대비 빌드업 |
| `[Chorus]` | 하이라이트, 감정 폭발, 캐치한 멜로디 | 가장 기억에 남는 구간 |
| `[Post-Chorus]` | 후렴 직후 여운, 에너지 전이 | 코러스 감성 연장 |
| `[Bridge]` | 분위기 환기, 곡의 전환점 | 새로운 관점 제시 |
| `[Outro]` | 여운을 남기며 마무리 | `[Fade Out]` 병용 가능 |

### v5 메타태그 인식 개선

> [!TIP]
> v5에서는 콜백 프레이징과 가사 마커(`[Verse]`, `[Chorus]`)의 인식 정확도가 크게 향상되었습니다.  
> 이전 버전에서 무시되거나 잘못 해석되던 복잡한 태그 조합도 안정적으로 처리합니다.

## 2. 장르별 보컬 및 감성 디렉팅

가사와 함께 보컬이 부를 스타일을 특정하고 싶다면, 파트 옆이나 가사 중간에 태그를 삽입할 수 있습니다.

### 보컬 발성 태그

| 태그 | 효과 |
|------|------|
| `[Whisper]` / `[Whispers]` | 속삭이는 보컬 |
| `[Spoken]` / `[Spoken Word Verse]` | 낭독/대사 |
| `[Shout]` / `[Belting]` | 외치기/벨팅 |
| `[Scream]` | 강렬한 샤우팅 |
| `[Falsetto]` | 가성 |
| `[Ad-lib]` | 즉흥 애드리브 |

### 랩/R&B 태그

- `[Flow]`, `[Fast Rap]`, `[Melodic Rap]` — 랩 스타일 지정
- `[Autotuned Delivery]` — 오토튠 질감

### 코러스/화음 표현

괄호를 사용하여 백보컬이나 더블링, 피처링 느낌을 줍니다:
```
나를 떠나지 마 (떠나지 마)
```

### v5 Persona 연동

> [!IMPORTANT]
> v5의 **Persona** 기능으로 보컬 캐릭터가 고정되면, 가사에서 매번 보컬 디렉팅 태그를 반복할 필요가 줄어듭니다.  
> Persona가 이미 보컬 톤·스타일을 정의하므로, 가사에서는 **특별히 변화가 필요한 구간**에만 보컬 태그를 삽입하면 됩니다.

**Persona 활용 예시:**
```
[Verse 1]
// Persona의 기본 톤으로 부드럽게 시작
달빛 아래 걸어가는 이 밤

[Chorus | Belting]
// 이 구간만 벨팅으로 감정 폭발
너를 잊지 못해 이 밤이 다 지나도
```

## 3. 기악 및 효과 태그 활용

특정 구간에서 보컬 없이 기악 연주가 부각되도록 하려면 악기 태그를 사용하세요.

### 솔로 연주

- `[Guitar Solo]`, `[Piano Solo]`, `[Synthesizer Solo]`, `[Drum Fill]`

### 분위기 전환

- `[Bass Drop]`, `[Beat Break]`, `[Modulation]`(조바꿈)
- `[Instrumental Bridge]`, `[Long Instrumental Break]` — 기악 구간 연장

### `//` 인라인 지시문 (v5 강화)

가사 중간에 `//`로 시작하는 줄을 삽입하면 **기술 지시**로 처리되며, AI가 가사로 부르지 않습니다:

```
[Bridge]
// Strip back to solo piano, vulnerable and intimate delivery
언젠가 다시 만날 수 있을까

// Gradually add strings, building emotion
그때까지 기다릴게
```

> v5에서는 `//` 주석의 해석 정확도가 향상되어, 악기 지정·에너지 변화·보컬 톤 변화 등의 지시를 더 세밀하게 반영합니다.

### 메타태그 스태킹

파이프 `|`로 복합 지시를 결합할 수 있습니다:
```
[Chorus | Harmonized Chorus | Female belting | Epic orchestral backing]
```

## 4. 가사의 작성 규칙과 팁

| 원칙 | 설명 |
|------|------|
| **호흡과 리듬** | 문장이 너무 길면 AI가 빠르게 부르거나 발음을 뭉뚱그림. 짧고 간결하게 작성 |
| **라임(Rhyme)** | 문장 끝 음절의 모음을 맞추면 더 자연스럽고 리듬감 있는 멜로디 생성 |
| **반복 구조** | `[Chorus]` 핵심 가사를 반복하면 중독성 있는 후크(Hook) 완성 |
| **괄호 활용** | `(가사)`로 백보컬·더블링·피처링 느낌 부여 |
| **감정 곡선** | 가사의 감정 흐름이 곡 구조의 에너지 곡선과 일치하도록 설계 |
| **v5 한국어** | v5는 한국어 가사 발음 정확도가 크게 개선. 단, 프롬프트는 영문 유지 |

### 앨범 단위 가사 일관성

Persona를 활용한 앨범 작업 시, 가사에서도 다음을 고려합니다:

- **같은 화자 유지** — 앨범 전체가 동일 인물의 시점인 경우, 1인칭 어조·어휘 수준 통일
- **테마 연결** — 곡 간 반복되는 키워드·이미지로 앨범의 서사적 연결감 확보
- **보컬 태그 최소화** — Persona가 보컬 톤을 고정하므로, 특별한 변화가 필요한 구간에만 태그 삽입

## 5. Studio Timeline & 가사 최적화

v5 Studio의 **섹션 기반 타임라인**에서는 각 섹션을 독립적으로 생성·교체할 수 있으므로, 가사도 섹션 단위로 최적화 가능합니다:

1. **섹션별 독립 가사 테스트** — 코러스 가사만 교체하여 여러 버전 비교
2. **Alternates 활용** — 동일 가사로 5~10개 대체 버전 생성, 멜로디·리듬 최적 테이크 선별
3. **레이어링** — `Add Vocals`로 하모니·백보컬 가사를 별도 레이어로 추가
